Anti-goals
==========

Purpose
-------

This document will capture what ``lifecore_state`` is not intended to become.
Sprint 17.8 will complete the anti-goals so later implementation work stays
bounded.

Initial anti-goals
------------------

- not a replacement for ROS 2 lifecycle nodes;
- not a hidden lifecycle manager;
- not an ECS runtime;
- not an EventBus framework;
- not a code generation system;
- not a broad runtime framework inside ``lifecore_ros2``;
- not a reason to add package metadata during Sprint 17.

Current scope
-------------

These anti-goals are review scaffolding. They do not create enforcement code,
runtime checks, package hooks, or public APIs.

Review questions
----------------

- Which anti-goals should be promoted to hard architecture constraints?
- Which anti-goals are Sprint 17-only constraints?
- Which anti-goals need examples to avoid future ambiguity?
