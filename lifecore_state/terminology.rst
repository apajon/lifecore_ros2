Terminology
===========

Purpose
-------

This document will define the shared vocabulary for ``lifecore_state``. Sprint
17.4 will complete the glossary before implementation details are proposed.

Planned terms
-------------

- observed state;
- command;
- state owner;
- state consumer;
- validity;
- freshness;
- lifecycle state;
- distributed state;
- message contract;
- package boundary.

Current scope
-------------

This is a documentation skeleton only. It does not define runtime classes,
message files, or public APIs.

Review questions
----------------

- Which terms must be stable before message semantics are reviewed?
- Which terms overlap with ROS 2 lifecycle vocabulary and need explicit
  separation?
- Which terms should remain informal until package boundaries are accepted?
