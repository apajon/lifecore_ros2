RFCs
====

Purpose
-------

This folder contains architecture RFC material for ``lifecore_state``. An RFC
records a proposed direction, the trade-offs behind it, and the review criteria
that must be settled before implementation begins.

Why RFCs first
--------------

``lifecore_state`` is intentionally separate from the existing
``lifecore_ros2`` lifecycle runtime. RFCs keep that separation explicit while
the architecture is still under review.

Sprint 17 scope
---------------

Sprint 17 RFC files are documentation artifacts only. They must not create ROS
2 packages, Python packages, build metadata, runtime modules, or compiled ROS 2
interfaces.
